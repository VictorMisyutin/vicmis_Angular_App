An Introduction to Vim for Beginners

Victor Misyutin

My first introduction to terminal text editors was using Nano to help set up my Ubuntu EC2 intance. The only reason I used Nano was because that was the text editor that the guy in the video was using.

It wasn't until about a month later that I even knew there were alternative to Nano. Mainly, Vim.

However, I never wanted to use Vim because unlike Nano, it seemed far more complex and difficult to use. I remember the first time I tried to use Vim instead of Nano I could not even figure out how to write to the file.

Then I watched YouTube video where some guy was using Vim and was literally making magic on my screen. He knew every shortcut and command and made Vim look extremly powerful. Many of those commands and shortcuts are not even possible using Nano, which is why I decided to start learning Vim.

What is Vim

Vim and Nano are both in-terminal text editors. Since they are terminal based that means that you can only interact with them using your keyboard, no mouse inputs. This is quite annoying but they are very useful when you SSH onto another system and need to edit and read files on that system.

How to Open Vim

To open Vim on a Linux system, first check if it is already installed. To do that run this command.

vim --version

If you get a message saying command not found, run:

apt install vim

If it prompts you to allocate space on your disk for this install press "y"

If you are on Windows or Mac you can find install directions here.

Vim Modes

Vim is essentially broken into seven main modes. Normal, visual-block, select, insert, command-line, ex, and terminal mode. However, I will mostly be focusing on four: normal, command, visual-block, and insert, which you will be using the most.

Normal Mode

Normal mod is the mode that you start in when Vim first opens. You cannot write directly to thie file in this mode because it is ment to be used for navigation and certain shortcuts. You can navigate the file in normal mode using your arrow key.

You can always navigate back to normal mode by pressing the escape key.

Insert Mode

Insert mode is the mode that you use to actually change the contents of the file. To enter insert mode press "i".

Once you are in insert mode you can write to the file as you would in and IDE or notepad. The only difference is that you some shortcuts dont work like they do in and IDE. For example, you cannot press "ctrl" + "x" to delete a line. Instead, you would have to navigate back to normal mode (using the escape key) and press "dd" quickly on the line you want to remove.

Visual-Block Mode

If you want to copy a few words or a phrase you will have to use the visual-block mode. To do this, navigate to where you want to start copying. Then press "ctrl" + "v" to enter visual-block mode. Then, use navigate to where you want the copy to end. You should see all of the text get selected. To copy that selected text press "y", which should copy the text and place you back in normal mode. Then, like before, press "p" wherever you want to paste

You can also use the visual-block mode to delete a string. Like before, use visual-block mode to select some text and then press "d" to cut.

Command-Line Mode

Command mode is used to enter other commands that make Vim more powerfull. To enter command mode you first have to be in normal mode. Once in normal mode you can press ":" to enter command mode.

Once you press ":" you will notice that at the bottom left of your screen you there is a small line that contains a ":" where you can enter your commands.

One such command is ":w" (write) which you can use to write to the file. There is also ":q" (quite) which you can used to quit vim.

An important thing to note is that if you add "!" to the end of a command, it overrides any errors or warning and runs the command anyway. For example, if you make changes to a file and then try to run ":q" without first saving using ":w" you will get a warning saying that you have not saved your changes. However, you can override this warning by running ":q!".

Other commands

Here I am just going to list a bunch of other commands that I use commonly.

As I mentioned before you can use ":w" to save the file and ":q" to quit the file. However, you can also you use

:wq

To simultaneously save and exit the file.

You can also use

:set number

or

:set nu

To display the line numbers on the left side of the screen.

To hide the numbers again you can run

:set nonumber

or

:set nonu

To navigate to the end of the file you can run

:$

To go to the start of a file you cand press "gg" (g two times quickly) to navigate back to the start of the file.

If you want to delete a line, in normal mode navigate to that line and press "dd" (d two times quickly) to delete that line.

If you want to copy a line in normal mode (whithou going to visual-block mode) navigate to that line and press "yy" to copy/yank that line. Then, again in normal mode navigate to where you want to paste that line and press "p" to paste. If you want to copy multiple lines after your cursor you can type in a number before pressing "yy". For example, entering "4yy" will copy/yank our lines.

In normal mode, if you want to start a new line you can press "o". Note that this will place you in a new line and move you into insert mode.

You can search for a word or phrase in your file by running the command

:/<string_you_want_to_search>

It is important to know that the string you are searching for is case sensative. That command will find the first instance of that if there is one. To find the next instance of that string you can press "n".

If you want to undo something, you can press "u" in normal mode

The last two commands I want to go over are "split" and "vplit". Both of these commands are used to display two or more files on one screen. If you run

:split<name_of_second_file>

It will open that second file above the file you already have open. These screens are seperate, so any input or command you do to one file will not affect the other. If you want to switch between the files you have to press "ctrl" + "ww".

When you first open the second file, both files will take up equal amounts of space on in the terminal. If you want to resize either file you can switch to that file and run

:resize <new_size>

Where 0 hides the file and the larger the number the bigger the file is displayed.

Alternatively, you can run

:vertical split <name_of_second_file>

This way, the files will display vertically side by side. If you prefer this, you have to resize the windows by using

:vertical resize <new_size>

I know that all of those command and modes might be overwhelming and seem to complicated but I urge you to try Vim. Just like anything you will eventually get used to all of those commands and shortcuts. Although you might be slower at first, Vim will save you a lot of time in the long run.
